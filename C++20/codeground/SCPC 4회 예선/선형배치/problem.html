선형 배치 (Linear Arrangement)

📝 문제 설명

$N$개의 정점으로 구성된 그래프 $G=(V,E)$가 있습니다. 각 정점은 $0$과 $N-1$ 사이의 번호로 구분됩니다.

모든 정점을 $x$축 상의 위치 $0, 1, \dots, N-1$에 하나씩 배치하여, 모든 간선의 길이 총합이 최소가 되도록 하는 문제입니다. 정점 $v$의 위치를 $P_v$라고 할 때, 두 정점 $u, v$를 잇는 간선의 거리는 $|P_u - P_v|$가 됩니다.

우리가 최소화해야 할 값은 $\sum_{(u,v) \in E} |P_u - P_v|$ 입니다.

예를 들어, 위 그래프를 특정 방식으로 배치하면 간선 길이 합이 16이 되지만, 더 효율적으로 배치하면 14가 될 수 있습니다.

⚙️ 제약 조건 및 채점
  • 시간 제한: 전체 테스트 케이스 4초 이내 (Java 8초 이내)
  • 메모리 제한: Heap, Global, Static 총 256MB, Stack 1MB
  • 점수 배분:
    • 그룹 1 (40점): $N < 9$인 경우. 이 그룹에서는 최적해를 찾아야 합니다.
    • 그룹 2 (160점): 별도 제한 없는 경우. 이 그룹에서는 출제자의 기준 해보다 좋거나 비슷한 해를 찾으면 점수를 받습니다.
  • 참고: 이 문제는 큰 N에 대해 최적해를 찾기보다 상대적으로 좋은 해를 찾는 휴리스틱(heuristic) 문제입니다.
📥 입력 형식

첫 줄에 테스트 케이스의 개수 $T$ ($1 \le T \le 30$)가 주어집니다.

각 테스트 케이스의 첫 줄에는 정점의 개수 $N$ ($5 \le N \le 100$)과 간선의 개수 $M$ ($4 \le M \le 1,000$)이 주어집니다.

다음 $M$개의 줄에 걸쳐 간선으로 연결된 두 정점 $u, v$ ($0 \le u, v \le N-1$)가 주어집니다.

📤 출력 형식

각 테스트 케이스마다 다음을 순서대로 출력합니다.

  1. "Case #C" (C는 1부터 시작하는 테스트 케이스 번호)
  2. 계산된 모든 간선의 최소 길이 합
  3. 좌표 $0$부터 $N-1$ 순서대로 배치된 정점의 번호 (공백으로 구분)
